# 龙虾日记 v2.0 多角色协作开发计划

## 🎯 协作模式

由于环境限制，采用**顺序协作**模式，每个角色依次输出规划：

```
产品经理 → 前端开发 → 后端开发 → 测试验收
    ↑                                    ↓
    └──────────── 迭代反馈 ←─────────────┘
```

---

## 👥 角色分工

| 角色 | 职责 | 输出 |
|------|------|------|
| 🎯 产品经理 | 需求分析、优先级规划 | PRD 文档 |
| 💻 前端开发 | UI/UX 实现、页面开发 | 代码 + 截图 |
| ⚙️ 后端开发 | API、数据库、部署 | API 文档 |
| 🧪 测试工程师 | 功能测试、Bug 修复 | 测试报告 |

## 🚀 开发流程

### Step 1: 产品规划
- 分析用户反馈
- 确定功能优先级
- 输出 PRD 文档

### Step 2: 前端开发
- 实现新功能 UI
- 优化交互体验
- 适配移动端

### Step 3: 后端开发
- 实现新 API
- 数据库优化
- 部署配置

### Step 4: 测试验收
- 功能测试
- 性能测试
- Bug 修复

